# Первичный Бульон — Техническое Задание (MVP)

## Концепция

Idle-roguelike с нейроэволюцией, вдохновлённый клеточным этапом Spore.
Игрок создаёт существо в редакторе, а нейросеть учится за него выживать.
Мир состоит из **слоёв** (уровней сложности). NN обучается выживать на текущем слое;
при достижении порога — переход на следующий, где сложнее, но есть новые запчасти.
Если за 50 поколений порог не пройден — откат, игрок дорабатывает существо.
За прохождение слоёв выдаются **гены** (валюта) — можно фармить любой слой без убывания наград.

## 1. Ядро: Физика и мир (без Matter.js)

### 1.1 Собственная мини-физика

Никаких внешних физических движков. Всё на простой кинематике:

```
position += velocity * dt
velocity *= (1 - drag * dt)
```

**Файл `core/vec2.js`** — 2D вектор:
- add, sub, scale, normalize, dot, cross, length, distanceTo
- rotate(angle), lerp(other, t), angle()
- Всё иммутабельно (возвращает новый) + мутабельные варианты (addSelf и т.д.)

**Файл `core/spatial.js`** — Spatial Hash Grid:
- Мир разбит на ячейки (напр. 128×128 px)
- Каждое тело вставляется в ячейку по позиции
- Запрос `query(x, y, radius)` → все тела в соседних ячейках
- Это заменяет Matter.Query.region, но в 10× быстрее
- Перестраивается каждый тик (insert all → query → clear)

**Файл `core/collision.js`** — Простые коллизии:
- Circle vs Circle: `dist < r1 + r2` → overlap, normal, pushback
- Circle vs Wall: bounce off bounds
- Коллизии только между **головами** (сегмент 0) существ + окружающие объекты
  (внутренние сегменты тела не коллайдятся с миром, только визуально следуют)
- Сенсорные коллизии (еда, яд) — попадание любого сегмента в радиус

### 1.2 Мир

**Файл `world/soup.js`**:
- `WORLD_SIZE` — квадратный мир (подбираем под слой, базово ~4096–8192)
- Стенки мира — мягкий pushback (не мгновенная смерть; но долгое стояние у стены — медленный урон)
- Еда — статичные точки с типом (растительная / мясо), радиусом, энергией
- Яды/шипы — зоны урона
- Простой тик: `physics → collisions → sensors → brains → apply forces → deaths → spawn`

## 2. Существо: Сегментное тело

### 2.1 Позвоночник (Verlet Chain)

**Файл `creature/body.js`**:

Существо = цепочка **сегментов** (3–8 штук для MVP):

```
[Голова] → [Сегмент 1] → [Сегмент 2] → ... → [Хвост]
```

Каждый сегмент:
```js
{
  pos: Vec2,          // мировая позиция
  radius: number,     // толщина в этой точке (редактируется игроком)
  parts: [],          // прикреплённые части (рот, глаз, плавник...)
}
```

**Движение (verlet follow):**
1. **Голова** двигается по `velocity` (управляется NN)
2. Каждый следующий сегмент **следует за предыдущим** с фиксированным расстоянием:
   ```
   dir = (prev.pos - curr.pos).normalize()
   curr.pos = prev.pos - dir * segmentSpacing
   ```
3. Это даёт естественный змейкообразный эффект без пружин и constraints

**Направление:**
- `heading` = угол вектора `segment[0].pos - segment[1].pos`
- Это «перед» существа. Все сенсоры, части, отрисовка привязаны к heading

### 2.2 Отрисовка (Smooth Snake)

**Файл `creature/render.js`**:

Тело рисуется **не как набор кругов**, а как гладкая замкнутая форма:

1. Для каждого сегмента вычислить два контурных point'а (left, right) перпендикулярно направлению:
   ```
   perp = direction.rotate(90°)
   left[i]  = segment[i].pos + perp * segment[i].radius
   right[i] = segment[i].pos - perp * segment[i].radius
   ```
2. Соединить left-точки от головы к хвосту, затем right-точки от хвоста к голове
3. Провести через все точки **сплайн** (quadratic bezier или catmull-rom) → гладкий контур
4. Залить цветом (с градиентом вдоль тела) → stroke тонкой линией

Результат: плавная «амёба-змейка», не набор шариков.

**Дополнительная отрисовка:**
- Прикреплённые части рисуются поверх контура в позициях сегментов
- Ядро (nucleus) — в центральном сегменте, маленький круг
- Глаза — на голове, направлены по heading

### 2.3 Редактор существа (Spore-style)

**Файл `game/editor.js`**:

Режим редактирования (по клавише E или между слоями):

**Редактирование толщины:**
- Игрок наводит мышь на сегмент
- Подсвеченный сегмент реагирует на **scroll**: ↑ толще, ↓ тоньше
- Мин/макс радиус: 5–30 px (масштабируется от общего размера существа)
- Визуально мгновенно видно как меняется форма контура

**Прикрепление частей:**
- Внизу — панель доступных частей (открытых на текущем слое)
- Drag-and-drop на конкретный сегмент
- У каждого сегмента максимум 2 part-слота (лево/право)
- Голова имеет дополнительный слот «спереди» (для рта)
- Хвост имеет дополнительный слот «сзади»

**Стоимость:**
- Каждая часть стоит N генов
- Текущий баланс генов показан
- Части можно снять (возврат генов)

## 3. Части тела (MVP набор)

### 3.1 Рот хищника (Carnivore Mouth)
- **Слот:** передний (только на голове)
- **Эффект:** может есть других существ (включая своих) и вражеских NPC
- **Механика:** при столкновении головой с другим существом — укус, наносит урон, получает энергию пропорционально урону
- **Несовместим** с усиками (нельзя иметь оба)
- **Визуал:** зубастый полукруг на голове

### 3.2 Усики / Реснички (Cilia / Antennae)
- **Слот:** передний (только на голове)
- **Эффект:** может есть растительную еду (траву/водоросли)
- **Механика:** при столкновении с едой — поглощает с хорошей усваиваемостью (100%)
- **Несовместим** с ртом хищника (нельзя иметь оба)
- **Визуал:** пара тонких усиков, шевелятся

### 3.3 Без рта/усиков (Omnivore по умолчанию)
- Если не установлен ни рот, ни усики → всеядный
- Может есть и мясо (при столкновении с телами), и траву
- **Усваиваемость крайне низкая** — получает лишь ~25% энергии
- Это намеренно плохой вариант — стимул выбрать специализацию

### 3.4 Плавники (Fins)
- **Слот:** боковой (любой сегмент, лево/право)
- **Эффект:** при «махе» — импульс ускорения в направлении движения головы
- **Механика NN:** 
  - Плавник добавляет **дополнительный выход** в NN: `fin_flap` (-1..1)
  - Положительное значение = мах → кратковременный буст скорости
  - Частые махи = больше расхода энергии
  - Если плавников несколько — каждый добавляет свой выход
- **Визуал:** анимированный плавник на боку сегмента, мах = визуальный поворот

### 3.5 Шип (Spike)
- **Слот:** боковой (любой сегмент)
- **Эффект:** при **высокой скорости** столкновение наносит урон другим существам (включая своих)
- **Механика:**
  - Урон = `base_damage * (speed / max_speed)`
  - Если скорость < порога — шип не наносит урон
  - Шип не расходует энергию (пассивный)
- **Визуал:** острый выступ на контуре тела, слегка покачивается

### 3.6 Глаза (Eyes)
- **Слот:** боковой (лучше на голове, но можно на любом сегменте)
- **Эффект:** увеличивают дальность сенсоров
- **Механика:**
  - Без глаз: `SENSOR_RANGE = 40 px` (почти слепой, чувствует только рядом)
  - Каждый глаз: `+60 px` к дальности
  - Максимум 2 глаза = 40 + 120 = 160 px дальности
  - Глаза также дают **дополнительные сенсорные лучи** (без глаз — 4 луча, с глазами — 8)
- **Визуал:** круглый глаз с зрачком, смотрит в направлении heading

## 4. Нейросеть (Составная NN)

### 4.1 Архитектура

NN — **модульная**: основной мозг + суб-модули для частей.

```
                    ┌──────────────┐
  Сенсоры ────────→ │  Core Brain   │──→ thrust, turn
  HP%, Energy% ──→ │  (всегда)     │──→ memory
                    └──────┬───────┘
                           │ hidden state
                    ┌──────┴───────┐
  (если есть fins)  │  Fin Module   │──→ fin_flap_1, fin_flap_2...
                    └──────────────┘
```

**Core Brain (всегда присутствует):**
- Входы: сенсорные лучи + HP% + Energy% + food_vec + danger_vec + memory
- Скрытый слой: 10–12 нейронов (tanh)
- Выходы: `thrust` (0..1), `turn` (-1..1), `memory` (-1..1)

**Part Modules (добавляются динамически):**
- При установке плавника → добавляется `FinModule`:
  - Вход: несколько нейронов из hidden state Core Brain
  - Свой маленький скрытый слой (3–4 нейрона)
  - Выход: `fin_flap` (-1..1)
- При добавлении второго плавника → второй FinModule с отдельными весами
- Модули мутируют и наследуются вместе с Core Brain

**Зачем модульность:**
- Когда игрок добавляет новую часть, Core Brain **сохраняет обученные веса**
- Новый модуль инициализируется случайно → NN быстро доучивает только его
- Без модульности: при добавлении части менялся бы размер сети, и все старые веса пропадали

### 4.2 Сенсоры (эгоцентричные)

**Файл `creature/sensors.js`**:

Все сенсоры — **относительно heading** (направления головы), не мировые:

```
Без глаз (4 луча):
       front
    ╱    |    ╲
  FL    FWD    FR
         |
        ---  (нет задних)

С 1 глазом (6 лучей):
  добавляются LEFT и RIGHT (90°)

С 2 глазами (8 лучей):
  добавляются BACK-LEFT и BACK-RIGHT
```

Каждый луч возвращает: `type_code * inverse_distance`
- Еда (трава): +1.0
- Еда (мясо/тело): +0.6
- Свои (другие существа вида): +0.2 (если рот хищника — -0.3)
- Яд: -1.0
- Шип-зона: -0.8
- Враг: -0.7
- Стена: -0.5

**Дополнительные входы:**
- `hp / maxHp` (0..1)
- `energy / maxEnergy` (0..1)
- `nearest_food_angle` (-1..1) — угол к ближайшей еде отн. heading
- `nearest_food_dist` (0..1) — расстояние (инвертированное)
- `nearest_danger_angle` (-1..1)
- `nearest_danger_dist` (0..1)
- `angular_velocity` (-1..1)
- `memory_prev` (-1..1) — рекуррентный нейрон из прошлого тика

**Итого входы (Core Brain):**
- 4–8 лучей (зависит от глаз) + 8 фиксированных = **12–16 входов**
- NN должна уметь работать с переменным числом входов → лучше фиксировать 8 лучей, но без глаз дальние заполнять нулями

### 4.3 Выходы

| Выход | Диапазон | Описание |
|-------|----------|----------|
| `thrust` | 0..1 | Сила движения вперёд (по heading) |
| `turn` | -1..1 | Поворот (- = влево, + = вправо) |
| `memory` | -1..1 | Записывается, возвращается в следующем тике |
| `fin_flap_N` | -1..1 | Мах N-го плавника (только если установлен) |

### 4.4 Эволюция

**Файл `brain/pool.js`**:

- Архив лучших геномов (200 шт)
- При смене поколения: top 20% → mutate + crossover → новая популяция
- Adaptive mutation: стагнация → сильнее мутации
- **Геном включает** веса Core Brain + все Part Modules
- При добавлении новой части в редакторе: Part Module добавляется с рандомными весами, Core Brain остаётся

### 5.2 Логика прогрессии

**Файл `game/progression.js`**:

```
Состояние:
  currentLayer: number
  genes: number              // валюта игрока
  unlockedParts: Set         // что уже открыто
  genomeArchive: object      // сохранённый лучший геном (между слоями)
  layerAttempts: number      // попыток на текущем слое
  thresholdStreak: number    // сколько поколений подряд порог выполнен
```

**Цикл:**
1. Начать слой → загрузить параметры → спавн мира + популяции
2. NN обучается (поколение за поколением, фоново или ускоренно)
3. Каждое поколение: проверить порог
   - Порог выполнен → `thresholdStreak++`
   - Не выполнен → `thresholdStreak = 0`
4. Если `thresholdStreak >= threshold.sustainGenerations`:
   - **СЛОЙ ПРОЙДЕН** → выдать гены, открыть части, перейти дальше
5. Если поколений > `maxGenerations` и порог не пройден:
   - **НЕУДАЧА** → показать экран "Адаптация не удалась"
   - Игрок может: переиграть слой / вернуться в редактор / перейти к прошлому слою фармить гены

**Фарм:** Игрок может вернуться на любой пройденный слой и переиграть его.
Награда в генах **не уменьшается** — фиксированная за каждое прохождение.
Это даёт возможность фармить гены, если на следующем слое не хватает прокачки.

│ Особей: 147/250         │  Гены: 230            │
│ Лучший фитнес: 42.3     │  Средний: 18.7        │
│                         │  Порог: 15.0 ✓ (2/3)  │
├─────────────────────────┤                        │

```

Снизу-справа: миникарта
Справа: панель выбранного существа (HP, энергия, фитнес, мозг)

```

### 6.3 Редактор существа

Полноэкранный оверлей. Существо показано крупно в центре:

- **Наведение на сегмент** → подсветка, scroll = толщина
- **Панель частей** внизу (доступные): drag на сегмент
- **Стоимость** каждой части в генах показана
- **Статы** справа: скорость, HP, дальность обзора, урон, усваиваемость
- Кнопки: [Применить] [Сбросить] [Назад]

### 6.4 Управление камерой

- **Drag** (ЛКМ зажата) = панорама
- **Scroll** = zoom in/out
- **Клик на существо** = фокус + показать инфо-панель
- Камера плавно следует за выбранным существом
- Двойной клик по пустоте = снять фокус

### 7.3 Еда

| Тип | Энергия | Усваиваемость (усики) | Усваиваемость (рот) | Усваиваемость (ничего) |
|-----|---------|----------------------|---------------------|----------------------|
| Трава | 20–30 | 100% | 0% (не может) | 25% |

### 7.4 Фитнес

```
fitness = age_seconds * (1 + food_eaten * 0.5)
        + food_eaten * 2.0
        - wall_penalty * 0.3
        - idle_penalty * 0.5
```

Порог слоя проверяет средний фитнес или % выживших (зависит от слоя).

## 10. Технические решения

### 10.1 Почему без Matter.js
- Наша физика тривиальна: позиция + скорость + drag + circle коллизии
- Matter.js добавляет ~300KB + рассчитывает ненужные rigid body constraints
- Verlet chain для тела — 20 строк, не нужен physics engine
- Spatial hash в 5–10× быстрее Matter.Query для наших запросов
- Легче делать headless-симуляцию (нет зависимости от DOM/browser API)

### 10.2 Модульная NN vs единая
- **Единая NN**: при добавлении плавника надо менять размер сети → все веса теряются
- **Модульная NN**: Core Brain фиксированного размера + Part Modules подключаются отдельно
- Core Brain всегда: 16 входов → 12 hidden → 3 выхода (thrust, turn, memory)
- Part Module: 4 входа (из hidden Core) → 4 hidden → 1 выход
- Геном = { core: {weights}, modules: [{type, weights}, ...] }

### 10.3 Headless Turbo
- Кнопка "Turbo ×50" — пропускает все draw(), только физика + NN
- Можно прогнать 50 поколений за минуту вместо часа
- В будущем: Web Worker для параллельных симуляций

### 10.4 Сохранение
- Всё состояние сериализуется в JSON: геном, гены, пройденные слои, конфиг тела
- Flask endpoints: `POST /api/save`, `GET /api/load/<name>`, `GET /api/saves`
- Опционально: localStorage для автосейва
